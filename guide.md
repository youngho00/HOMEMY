# 포트폴리오 사이트 기능 추가 가이드

요청하신 **이메일 문의 기능(Contact)**과 **데이터베이스 없는 자유게시판(Board)**이 성공적으로 구현되었습니다. 아래는 각 기능의 구성 방식과 사용 방법입니다.

## 1. 구현된 주요 기능

### 📬 이메일 문의 (Contact)
- **위치**: `contact.html` 및 메인 페이지 우측 하단 플로팅 버튼
- **작동 방식**: 사용자가 이름, 이메일, 제목, 내용을 입력하고 '메시지 보내기'를 누르면 브라우저의 전송 기능을 사용하여 사용자의 이메일 클라이언트(Outlook, Mail 등)가 자동으로 열립니다.
- **주요 코드**: `mailto:abcd1234@gmail.com`를 사용하여 백엔드 없이도 메일을 보낼 수 있도록 구성했습니다.

### 📋 자유게시판 (Board)
- **위치**: `board.html`
- **작동 방식**: 브라우저의 저장 공간인 `localStorage`를 사용합니다. 별도의 데이터베이스 서버 없이도 사용자가 작성한 글이 해당 브라우저에 저장되어 다시 방문했을 때도 유지됩니다.
- **특징**: 이름, 이메일(비공개), 제목, 내용을 입력하여 게시글을 남길 수 있으며, 본인이 작성한 글을 삭제할 수 있는 기능도 포함되어 있습니다.

## 2. 사용자 가이드 및 커스터마이징

### 📧 이메일 주소 변경하기
현재 메일 주소는 `abcd1234@gmail.com`으로 설정되어 있습니다. 본인의 이메일로 바꾸려면 다음 파일들을 수정하세요.

1. **`contact.html`**:
   - 하단 `<script>` 부분의 `mailto:abcd1234@gmail.com` 부분을 본인 이메일로 변경.
   - 하단 푸터 영역(`contact-info-footer`)의 텍스트 변경.

### 🏠 메인 페이지 연동
- 메인 페이지(`index88.html`) 우측 하단에 **"요청사항 보내기"** 플로팅 버튼이 추가되었습니다. 이 버튼은 클릭 시 즉시 `contact.html`로 연결됩니다.
- 상단 네비게이션 바에 **Board**와 **Contact** 메뉴가 추가되어 편리하게 이동할 수 있습니다.

### 💡 주의사항 (데이터베이스 미사용 관련)
- 현재 게시판은 `localStorage` 방식이므로 **글을 쓴 브라우저에서만** 목록이 보입니다. (다른 사람이 쓴 글을 서버에서 받아오는 기능은 데이터베이스 서버가 필요합니다.)
- 실시간으로 모든 사용자의 글을 공유하고 싶다면 **Firebase**나 **Supabase** 같은 무료 DB 서비스를 연동하는 것을 추천드립니다.
